target_compile_features

Новое в версии 3.1.

Добавить ожидаемые возможности компилятора к цели.

target_compile_features(<target> <PRIVATE|PUBLIC|INTERFACE> <feature> [...])

Указывает возможности компилятора, необходимые при компиляции данной цели. Если функция не указана в переменных CMAKE_C_COMPILE_FEATURES, CMAKE_CUDA_COMPILE_FEATURES или CMAKE_CXX_COMPILE_FEATURES, CMake сообщит об ошибке. Если использование функции требует дополнительного флага компилятора, такого как -std=gnu++11, флаг будет добавлен автоматически.

Ключевые слова INTERFACE, PUBLIC и PRIVATE требуются для указания области применения функций. Элементы PRIVATE и PUBLIC будут заполнять свойство COMPILE_FEATURES цели <target>. Элементы PUBLIC и INTERFACE будут заполнять свойство INTERFACE_COMPILE_FEATURES цели <target>. Повторные вызовы для той же цели <target> добавляют элементы.

Новое в версии 3.11: Позволяет задавать элементы INTERFACE для импортируемых целей.

Указанная цель <target> должна быть создана командой, такой как add_executable() или add_library(), и не должна быть целью-алиасом.

Для получения дополнительной информации о задании свойств системы сборки.

Аргументы к target_compile_features могут использовать выражения генератора с синтаксисом $<...>. См. руководство cmake-generator-expressions(7) для доступных выражений. См. руководство cmake-compile-features(7) для информации о функциях компиляции и списка поддерживаемых компиляторов.
