add_executable

Добавление исполняемого файла в проект с использованием указанных исходных файлов.

Обычные исполняемые файлы

add_executable(<name> <options>... <sources>...)

Добавляет цель исполняемого файла с именем <name>, который будет создан из указанных в команде исходных файлов.

Опции:

  • WIN32 автоматически устанавливает свойство цели WIN32_EXECUTABLE. Подробности смотрите в документации по этому свойству цели.
  • MACOSX_BUNDLE автоматически устанавливает свойство цели MACOSX_BUNDLE. Подробности смотрите в документации по этому свойству цели.
  • EXCLUDE_FROM_ALL автоматически устанавливает свойство цели EXCLUDE_FROM_ALL. Подробности смотрите в документации по этому свойству цели.

Имя <name> соответствует логическому имени цели и должно быть уникальным в рамках проекта. Фактическое имя созданного исполняемого файла формируется в соответствии с конвенциями родной платформы (например, <name>.exe или просто <name>).

Новое в версии 3.1: Аргументы исходных файлов для add_executable могут использовать "генераторные выражения" с синтаксисом $<...>. См. руководство по cmake-generator-expressions(7) для доступных выражений.

Новое в версии 3.11: Исходные файлы могут быть опущены, если они добавляются позже с помощью target_sources().

По умолчанию исполняемый файл будет создан в каталоге дерева сборки, соответствующем каталогу дерева исходных файлов, в котором была вызвана команда. См. документацию по свойству цели RUNTIME_OUTPUT_DIRECTORY для изменения этого местоположения. См. документацию по свойству цели OUTPUT_NAME для изменения части <name> итогового имени файла.

См. руководство по cmake-buildsystem(7) для получения дополнительной информации о определении свойств системы сборки.

Также см. HEADER_FILE_ONLY, что делать, если некоторые исходные файлы предварительно обрабатываются, и вы хотите, чтобы оригинальные исходные файлы были доступны из среды разработки (IDE).

Импортированные исполняемые файлы

add_executable(<name> IMPORTED [GLOBAL])

Добавляет цель импортированного исполняемого файла для ссылки на исполняемый файл, находящийся за пределами проекта. Имя цели может использоваться как любая цель, созданная в проекте, за исключением того, что по умолчанию она видима только в каталоге, в котором она создана, и в подкаталогах.

Опции:

  • GLOBAL делает имя цели видимым глобально.

Для импортированных целей не создаются правила сборки, и свойство цели IMPORTED равно True. Импортированные исполняемые файлы удобны для ссылок из таких команд, как add_custom_command().

Подробности об импортированном исполняемом файле указываются путем установки свойств, имена которых начинаются с IMPORTED_. Наиболее важным таким свойством является IMPORTED_LOCATION (и его версия для конфигурации IMPORTED_LOCATION_), которое указывает местоположение основного исполняемого файла на диске. См. документацию по свойствам IMPORTED_* для получения дополнительной информации.

Псевдонимы исполняемых файлов

add_executable(<name> ALIAS )

Создает цель-псевдоним, так что <name> можно использовать для ссылки на <target> в последующих командах. <name> не появляется в сгенерированной системе сборки как цель make. <target> не может быть псевдонимом.

Новое в версии 3.11: Псевдоним может ссылаться на глобальную импортированную цель.

Новое в версии 3.18: Псевдоним может ссылаться на не глобальную импортированную цель. Такой псевдоним ограничен каталогом, в котором он создан, и подкаталогами. Свойство цели ALIAS_GLOBAL может использоваться для проверки, является ли псевдоним глобальным или нет.

Цели-псевдонимы могут использоваться в качестве целей для чтения свойств, исполняемых файлов для пользовательских команд и пользовательских целей. Их также можно проверять на наличие с помощью подкоманды if(TARGET). <name> не может использоваться для изменения свойств <target>, т.е. не может использоваться в качестве операнда команд set_property(), set_target_properties(), target_link_libraries() и т.д. Цель-псевдоним не может быть установлена или экспортирована.
