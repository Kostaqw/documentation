  • add_subdirectory

Добавление подкаталога в сборку

add_subdirectory(source_dir [binary_dir] [EXCLUDE_FROM_ALL] [SYSTEM])

Добавляет подкаталог в сборку. source_dir указывает каталог, в котором находятся файлы CMakeLists.txt и исходные файлы кода. Если это относительный путь, он будет оценен относительно текущего каталога (типичное использование), но он также может быть абсолютным путем. binary_dir указывает каталог, в который будут помещены выходные файлы. Если это относительный путь, он будет оценен относительно текущего каталога вывода, но он также может быть абсолютным путем. Если binary_dir не указан, будет использовано значение source_dir до расширения любого относительного пути (типичное использование). Файл CMakeLists.txt в указанном каталоге исходных файлов будет немедленно обработан CMake до того, как обработка в текущем входном файле продолжится после этой команды.

Если аргумент EXCLUDE_FROM_ALL предоставлен, то свойство EXCLUDE_FROM_ALL будет установлено для добавленного каталога. Это исключит каталог из сборки по умолчанию. См. полное описание свойства каталога EXCLUDE_FROM_ALL.

Новое в версии 3.25: Если предоставлен аргумент SYSTEM, то свойство каталога SYSTEM для подкаталога будет установлено в значение true. Это свойство используется для инициализации свойства SYSTEM каждой неимпортированной цели, созданной в этом подкаталоге.
