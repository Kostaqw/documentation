project

Установите имя проекта

project(<PROJECT-NAME> [<language-name>...])
    project(<PROJECT-NAME>
            [VERSION <major>[.<minor>[.<patch>[.<tweak>]]]]
            [DESCRIPTION <project-description-string>]
            [HOMEPAGE_URL <url-string>]
            [LANGUAGES <language-name>...])

Устанавливает имя проекта и сохраняет его в переменной PROJECT_NAME. При вызове из верхнего уровня CMakeLists.txt также сохраняет имя проекта в переменной CMAKE_PROJECT_NAME.

Также устанавливает переменные:

  • PROJECT_SOURCE_DIR, <PROJECT-NAME>_SOURCE_DIR — Абсолютный путь к исходному каталогу проекта.
  • PROJECT_BINARY_DIR, <PROJECT-NAME>_BINARY_DIR — Абсолютный путь к каталогу бинарных файлов проекта.
  • PROJECT_IS_TOP_LEVEL, <PROJECT-NAME>_IS_TOP_LEVEL — Новое в версии 3.21. Булево значение, указывающее, является ли проект верхнего уровня.

Дополнительные переменные устанавливаются опциональными аргументами, описанными ниже. Если какой-либо из этих аргументов не используется, то соответствующие переменные устанавливаются в пустую строку.

Опции

  • VERSION <version> — Опционально; не может быть использовано, если политика CMP0048 не установлена в NEW. Принимает аргумент <version>, состоящий из неотрицательных целых компонентов, т.е. <major>[.<minor>[.<patch>[.<tweak>]]], и устанавливает переменные:
    • PROJECT_VERSION, <PROJECT-NAME>_VERSION
    • PROJECT_VERSION_MAJOR, <PROJECT-NAME>_VERSION_MAJOR
    • PROJECT_VERSION_MINOR, <PROJECT-NAME>_VERSION_MINOR
    • PROJECT_VERSION_PATCH, <PROJECT-NAME>_VERSION_PATCH
    • PROJECT_VERSION_TWEAK, <PROJECT-NAME>_VERSION_TWEAK
    Новое в версии 3.12: Когда команда project() вызывается из верхнего уровня CMakeLists.txt, версия также сохраняется в переменной CMAKE_PROJECT_VERSION.
  • DESCRIPTION <project-description-string> — Новое в версии 3.9. Опционально. Устанавливает переменные:
    • PROJECT_DESCRIPTION, <PROJECT-NAME>_DESCRIPTION
    в <project-description-string>. Рекомендуется, чтобы это описание было относительно короткой строкой, обычно не более нескольких слов. Когда команда project() вызывается из верхнего уровня CMakeLists.txt, описание также сохраняется в переменной CMAKE_PROJECT_DESCRIPTION. Новое в версии 3.12: Добавлена переменная <PROJECT-NAME>_DESCRIPTION.
  • HOMEPAGE_URL <url-string> — Новое в версии 3.12. Опционально. Устанавливает переменные:
    • PROJECT_HOMEPAGE_URL, <PROJECT-NAME>_HOMEPAGE_URL
    в <url-string>, которая должна быть каноническим домашним URL для проекта. Когда команда project() вызывается из верхнего уровня CMakeLists.txt, URL также сохраняется в переменной CMAKE_PROJECT_HOMEPAGE_URL.
  • LANGUAGES <language-name>... — Опционально. Может быть также указано без ключевого слова LANGUAGES согласно первому, короткому синтаксису. Выбирает, какие языки программирования необходимы для сборки проекта. Поддерживаемые языки: C, CXX (т.е. C++), CSharp (т.е. C#), CUDA, OBJC (т.е. Objective-C), OBJCXX (т.е. Objective-C++), Fortran, HIP, ISPC, Swift, ASM, ASM_NASM, ASM_MARMASM, ASM_MASM и ASM-ATT. Новое в версии 3.8: Добавлена поддержка CSharp и CUDA. Новое в версии 3.15: Добавлена поддержка Swift. Новое в версии 3.16: Добавлена поддержка OBJC и OBJCXX. Новое в версии 3.18: Добавлена поддержка ISPC. Новое в версии 3.21: Добавлена поддержка HIP. Новое в версии 3.26: Добавлена поддержка ASM_MARMASM. Если включается ASM, укажите его последним, чтобы CMake мог проверить, работают ли компиляторы для других языков, таких как C, с ассемблером. По умолчанию C и CXX включены, если не указаны никакие опции языка. Укажите язык NONE, или используйте ключевое слово LANGUAGES и не указывайте языков, чтобы пропустить включение любых языков.

Переменные, установленные через опции VERSION, DESCRIPTION и HOMEPAGE_URL, предназначены для использования в качестве значений по умолчанию в метаданных пакета и документации.

Внедрение кода

Ряд переменных может быть определен пользователем для указания файлов, которые нужно включить на разных этапах выполнения команды project(). Ниже приведены шаги, выполняемые во время вызова project():

  • Новое в версии 3.15: Для каждого вызова project(), независимо от имени проекта, включите файл(ы) и модуль(и), указанные в CMAKE_PROJECT_INCLUDE_BEFORE, если установлены.
  • Новое в версии 3.17: Если команда project() указывает <PROJECT-NAME> как имя проекта, включите файл(ы) и модуль(и), указанные в CMAKE_PROJECT__INCLUDE_BEFORE, если установлены.
  • Установите различные переменные, относящиеся к проекту, как описано в разделах "Синопсис" и "Опции".
  • Только для самого первого вызова project():
    • Если CMAKE_TOOLCHAIN_FILE установлен, прочтите его как минимум один раз. Он может быть прочитан несколько раз и может быть прочитан снова при включении языков позже (см. ниже).
    • Установите переменные, описывающие хост и целевые платформы. Языко-специфические переменные могут быть установлены или не установлены на этом этапе. При первом запуске единственные языко-специфические переменные, которые могут быть определены, это те, которые могли быть установлены файлом инструментальной цепочки. При последующих запусках могут быть установлены языко-специфические переменные, кэшированные с предыдущего запуска.
    • Новое в версии 3.24: Включите каждый файл, указанный в CMAKE_PROJECT_TOP_LEVEL_INCLUDES, если установлен. Переменная игнорируется CMake после этого.
    • Включите любые указанные языки в вызове или языки по умолчанию, если они не были указаны. Файл инструментальной цепочки может быть прочитан повторно при включении языка в первый раз.
    • Новое в версии 3.15: Для каждого вызова project(), независимо от имени проекта, включите файл(ы) и модуль(и), указанные в CMAKE_PROJECT_INCLUDE, если установлены.
    • Если команда project() указывает <PROJECT-NAME> как имя проекта, включите файл(ы) и модуль(и), указанные в CMAKE_PROJECT__INCLUDE, если установлены.

Использование

Файл верхнего уровня CMakeLists.txt для проекта должен содержать прямой вызов команды project(). Включение его через команду include() недостаточно. Если такого вызова не существует, CMake выдаст предупреждение и сделает вид, что в верхнем уровне есть проект (Project) для включения языков по умолчанию (C и CXX).

Примечание: Вызовите команду project() в начале верхнего уровня CMakeLists.txt, но после вызова cmake_minimum_required(). Важно установить версии и настройки политики до вызова других команд, поведение которых они могут изменить, и по этой причине команда project() выдаст предупреждение, если этот порядок не соблюден. Также см. политику CMP0000.
