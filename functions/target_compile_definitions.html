target_compile_definitions

Добавляет определения компиляции к целевому объекту.

target_compile_definitions(<target>
          <INTERFACE|PUBLIC|PRIVATE> [items1...]
          [<INTERFACE|PUBLIC|PRIVATE> [items2...] ...])

Указывает определения компиляции, которые будут использоваться при компиляции указанного <target>. Указанный <target> должен быть создан с помощью команды, такой как add_executable() или add_library(), и не должен быть целевым объектом ALIAS.

Ключевые слова INTERFACE, PUBLIC и PRIVATE необходимы для указания области действия следующих аргументов. Элементы PRIVATE и PUBLIC будут заполнять свойство COMPILE_DEFINITIONS объекта <target>. Элементы PUBLIC и INTERFACE будут заполнять свойство INTERFACE_COMPILE_DEFINITIONS объекта <target>. Следующие аргументы указывают определения компиляции. Повторные вызовы для одного и того же <target> добавляют элементы в порядке вызова.

Новое в версии 3.11: Разрешено устанавливать элементы INTERFACE для целевых объектов IMPORTED.

Аргументы для target_compile_definitions могут использовать генераторные выражения с синтаксисом $<...>. См. руководство cmake-generator-expressions(7) для доступных выражений. См. руководство cmake-buildsystem(7) для получения дополнительной информации о определении свойств системы сборки.

Любой ведущий -D в элементе будет удален. Пустые элементы игнорируются. Например, следующие команды эквивалентны:

target_compile_definitions(foo PUBLIC FOO)
        target_compile_definitions(foo PUBLIC -DFOO)  # -D удален
        target_compile_definitions(foo PUBLIC "" FOO) # "" игнорируется
        target_compile_definitions(foo PUBLIC -D FOO) # -D становится "", затем игнорируется

Определения могут иметь значения:

target_compile_definitions(foo PUBLIC FOO=1)

Обратите внимание, что многие компиляторы трактуют -DFOO как эквивалент -DFOO=1, но другие инструменты могут не распознавать это во всех случаях (например, IntelliSense).
