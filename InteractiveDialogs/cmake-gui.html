cmake-gui

Синтаксис

cmake-gui [<options>]
cmake-gui [<options>] -B <path-to-build> [-S <path-to-source>]
cmake-gui [<options>] <path-to-source | path-to-existing-build>
cmake-gui [<options>] --browse-manual [<filename>]

Описание

Выполняемый файл cmake-gui представляет собой графический интерфейс CMake. Настройки конфигурации проекта могут быть указаны в интерактивном режиме. Краткие инструкции отображаются в нижней части окна во время работы программы.

CMake — это генератор систем сборки, работающий на разных платформах. Проекты определяют свой процесс сборки с помощью независимых от платформы CMake-файлов, которые включены в каждую директорию дерева исходного кода и имеют название CMakeLists.txt. Пользователи создают проект, используя CMake для генерации системы сборки для нативного инструмента на их платформе.

Опции

  • -S <path-to-source>
    Путь к корневому каталогу проекта CMake, который нужно собрать.
  • -B <path-to-build>
    Путь к каталогу, который CMake будет использовать в качестве корня каталога сборки.
    Если каталог не существует, CMake создаст его.
  • --preset=<preset-name>
    Имя предустановки, которую нужно использовать из файлов предустановок проекта, если они существуют.
  • --browse-manual [<filename>]
    Открывает справочник CMake в браузере и сразу завершает работу. Если указан <filename>, открывает этот файл внутри справочника вместо index.html.
  • -version [<file>], --version [<file>], /V [<file>]
    Показывает баннер с названием программы и версией, затем завершает работу. Вывод сохраняется в указанный <file>, если он указан.
  • -h, -H, --help, -help, -usage, /?
    Печатает справочную информацию по использованию и завершает работу.
    Описание использования включает интерфейс командной строки и её опции.
  • --help <keyword> [<file>]
    Печатает справку по одному из ключевых слов CMake.
    <keyword> может быть свойством, переменной, командой, политикой, генератором или модулем.
    Соответствующая справочная запись выводится в удобочитаемом текстовом формате. Вывод сохраняется в указанный <file>, если он указан.
    Изменено в версии 3.28: до CMake 3.28 эта опция поддерживала только имена команд.
  • --help-full [<file>]
    Печатает все справочные руководства и завершает работу.
    Все руководства выводятся в удобочитаемом текстовом формате. Вывод сохраняется в указанный <file>, если он указан.
  • --help-manual <man> [<file>]
    Печатает одно справочное руководство и завершает работу.
    Указанное руководство выводится в удобочитаемом текстовом формате. Вывод сохраняется в указанный <file>, если он указан.
  • --help-manual-list [<file>]
    Список доступных справочных руководств и завершение работы.
    Список содержит все руководства, для которых можно получить справку с помощью опции --help-manual, за которой следует имя руководства. Вывод сохраняется в указанный <file>, если он указан.
  • --help-command <cmd> [<file>]
    Печатает справку по одной команде и завершает работу.
    Запись из справочника cmake-commands(7) для команды <cmd> выводится в удобочитаемом текстовом формате. Вывод сохраняется в указанный <file>, если он указан.
  • --help-command-list [<file>]
    Список команд, для которых доступна справка, и завершение работы.
    Список содержит все команды, для которых можно получить справку с помощью опции --help-command, за которой следует имя команды. Вывод сохраняется в указанный <file>, если он указан.
