CMAKE_INSTALL_PREFIX

CMAKE_INSTALL_PREFIX каталог установки, используемый install().

Если make install вызывается или INSTALL создается, этот каталог добавляется ко всем каталогам установки.

По умолчанию эта переменная имеет следующий вид:

  • Новое в версии 3.29: Если CMAKE_INSTALL_PREFIX установлена переменная окружения, ее значение используется по умолчанию для этой переменной.
  • c:/Program Files/${PROJECT_NAME} для Windows.
  • /usr/local для платформ UNIX.

Смотрите CMAKE_INSTALL_PREFIX_INITIALIZED_TO_DEFAULT, как проект может выбрать собственное значение по умолчанию.

В UNIX можно использовать DESTDIR механизм для переноса всей установки в промежуточную область. Смотрите DESTDIR Переменную среды для получения дополнительной информации.

Установочный префикс также добавлен в CMAKE_SYSTEM_PREFIX_PATH, так что find_package(), find_program(), find_library(), find_path(), find_file() будет выполняться поиск по префиксу другого программного обеспечения. Это поведение можно отключить, установив CMAKE_FIND_NO_INSTALL_PREFIX значение TRUE перед первым project() вызовом.

Примечание: Используйте GNUInstallDirs модуль для предоставления опций в стиле GNU для расположения каталогов в рамках установки.

CMAKE_INSTALL_PREFIX может быть определен при настройке дерева сборки для установки его установочного префикса. Или, при использовании cmake(1) режима инструмента командной строки --install, можно указать другой префикс с помощью --prefix опции:

cmake --install . --prefix /my/install/prefix
