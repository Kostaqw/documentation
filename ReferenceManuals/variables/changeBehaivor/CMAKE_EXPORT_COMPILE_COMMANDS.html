CMAKE_EXPORT_COMPILE_COMMANDS

Новое в версии 3.5.

Включает или отключает вывод команд компиляции во время генерации.

Если включено, генерируется compile_commands.json файл, содержащий точные вызовы компилятора для всех единиц перевода проекта в машиночитаемой форме. Формат файла JSON выглядит следующим образом: 

[
  {
    "directory": "/home/user/development/project",
    "command": "/usr/bin/c++ ... -c ../foo/foo.cc",
    "file": "../foo/foo.cc",
    "output": "../foo.dir/foo.cc.o"
  },

                
...
                
  {
    "directory": "/home/user/development/project",
    "command": "/usr/bin/c++ ... -c ../foo/bar.cc",
    "file": "../foo/bar.cc",
    "output": "../foo.dir/bar.cc.o"
  }
]

Эта опция инициализируется переменной окружения CMAKE_EXPORT_COMPILE_COMMANDS и устанавливает свойство EXPORT_COMPILE_COMMANDS для всех целевых объектов.

Примечание: Эта опция реализована только генераторами Makefile и Ninja. В других генераторах она игнорируется.

В настоящее время этот параметр плохо работает в сочетании со свойством целевого объекта UNITY_BUILD или переменной CMAKE_UNITY_BUILD.
