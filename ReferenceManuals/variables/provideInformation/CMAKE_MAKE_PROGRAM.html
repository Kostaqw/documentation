CMAKE_MAKE_PROGRAMM

Инструмент, который может запускать нативную систему сборки. Значение может быть полным путём к исполняемому файлу или просто именем инструмента, если предполагается, что он находится в PATH.

Выбор инструмента зависит от CMAKE_GENERATOR, используемого для конфигурации проекта:

  • Makefile Generators устанавливают это значение в make, gmake или генератор-специфический инструмент (например, nmake для NMake Makefiles).
    • Эти генераторы сохраняют CMAKE_MAKE_PROGRAM в кэше CMake, чтобы его мог редактировать пользователь.
  • Ninja generator устанавливает это значение в ninja.
    • Этот генератор также сохраняет CMAKE_MAKE_PROGRAM в кэше CMake для редактирования пользователем.
  • Xcode generator устанавливает это значение в xcodebuild.
    • Этот генератор предпочитает искать инструмент сборки во время сборки, а не сохранять CMAKE_MAKE_PROGRAM в кэше CMake заранее, так как xcodebuild легко найти.
  • Для совместимости с версиями CMake до 3.2, если пользователь или проект явно добавляют CMAKE_MAKE_PROGRAM в кэш CMake, то CMake будет использовать указанное значение.
  • Visual Studio Generators устанавливают это значение на полный путь к MSBuild.exe (VS >= 10), devenv.com (VS 7,8,9) или VCExpress.exe (VS Express 8,9).
    • Эти генераторы предпочитают искать инструмент сборки во время сборки, а не сохранять CMAKE_MAKE_PROGRAM в кэше CMake заранее, так как инструменты зависят от версии и могут быть найдены с использованием реестра Windows. Это также необходимо, так как правильный инструмент сборки может зависеть от содержимого проекта (например, плагин Intel Fortran для VS 10 и 11 требует devenv.com для сборки файлов .vfproj, даже если MSBuild.exe обычно предпочтителен для поддержки CMAKE_GENERATOR_TOOLSET).
  • Для совместимости с версиями CMake до 3.0, если пользователь или проект явно добавляют CMAKE_MAKE_PROGRAM в кэш CMake, то CMake будет использовать указанное значение, если это возможно.
  • Green Hills MULTI generator устанавливает это значение на полный путь к gbuild.exe (Windows) или gbuild (Linux) в зависимости от используемого набора инструментов.

Как только генератор инициализирует определённое значение для этой переменной, изменение этого значения приведёт к неопределённому поведению.

Переменная CMAKE_MAKE_PROGRAM установлена для использования в коде проекта. Значение также используется инструментами cmake --build и ctest --build-and-test для запуска нативного процесса сборки.
